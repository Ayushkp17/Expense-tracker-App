# Expense Tracker

A production-ready Flutter mobile application for tracking personal expenses with Firebase authentication, real-time Firestore storage, and interactive spending analytics.

---

## Features

- **Secure Authentication** — Email/password login and signup with full validation. One unique account per email enforced by Firebase Auth.
- **Expense Management** — Add, edit, and delete expenses (amount, category, date, optional note).
- **Category Color Coding** — 11 pre-built categories, each with a distinct deterministic color.
- **Analytics Dashboard** — Interactive pie chart (by category) and bar chart (daily spending).
- **Real-time Sync** — All data persisted in Cloud Firestore, synced in real-time across sessions.
- **Month Navigation** — Browse expenses by month with a month selector.
- **Dark Mode** — Beautiful Material 3 dark theme with vibrant accent colors.
- **Responsive** — Adapts to phone, tablet, and large screen sizes.

## Setup

### 1. Clone or open the project

```bash
cd Expense-tracker-App
```

### 2. Configure Firebase

If you haven't already generated `firebase_options.dart`:

```bash
firebase login
dart pub global run flutterfire_cli:flutterfire configure
```

Select your Firebase project and target platforms (Android, iOS).

### 3. Enable Firebase services

In the [Firebase Console](https://console.firebase.google.com/):

1. **Authentication** → Sign-in method → Enable **Email/Password**
2. **Firestore Database** → Create database → Start in **Native mode**

### 4. Set Firestore Security Rules

In Firestore → Rules, use:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/expenses/{expenseId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

### 5. Install dependencies

```bash
flutter pub get
```

### 6. Run the app

```bash
# Android
flutter run

# iOS (requires Xcode on macOS)
flutter run -d ios

# Specific device
flutter devices
flutter run -d <device_id>
```

## Architecture Notes

- **State Management**: Riverpod 3 with `Notifier` / `NotifierProvider` API. All async operations use `AsyncValue.guard` for consistent loading/error propagation.
- **Data Isolation**: Expenses are scoped to each user via Firestore path `users/{uid}/expenses/{id}`. Firestore rules enforce this at the backend.
- **Real-time**: `watchExpenses` returns a `Stream<List<Expense>>` that updates the UI instantly on any Firestore change.
- **Auth Guards**: `GoRouter.redirect` checks Firebase auth state on every navigation to prevent unauthenticated access.
